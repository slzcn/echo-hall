# 回声厅 · Echo Hall

> 赛博风格的实时聊天空间 · 进来即聊 · 说话 · 留存

一个零门槛的匿名实时聊天室:打开即用、无需注册就能以随机身份进入畅聊,也可注册正式账号跨设备保留身份与记录。全站基于 **Supabase**(匿名登录 + RLS + Realtime),前端是单文件静态页面,可直接部署到任意静态托管。

## ✨ 玩法一览

### 房间
- **官方频道** — 内置的常驻聊天频道
- **公开房** — 任何人可创建、大厅可见、谁都能进;最后一人离开自动归档
- **私密房** — 仅凭邀请码进入(10 位、约 50 bit 熵,抗枚举暴破)

### 身份
- 进场随机生成「形容词 + 动物 + emoji + 颜色」的临时身份,可一键换一个
- 可升级为正式账号(用户名 + 密码,选填邮箱找回),匿名期间的历史消息一并继承
- 已注册邮箱的账号支持跨设备登录与找回密码

### 互动
- **回声 🔥** — 给别人的消息点赞,实时同步、带粒子特效(不能给自己点)
- **引用回复 ↳** — 长按/右键消息发起引用,对方也能看到原文
- **投影 📽️** — 把一句话做成全屏飞幕弹幕,广播给全房间
- **悄悄话** — `/whisper @昵称 内容`,仅目标可见(RLS 双重拦截)
- **共鸣 ✨** — 短时间内多人发同一个 emoji 会触发全屏共鸣特效
- **正在输入 / 在线光墙** — 实时呈现谁在房、谁在打字

### 🎤 语音消息
点输入区的麦克风按钮即可录音发送(最长 30 秒),消息流里渲染成自绘播放器(播放/暂停 + 动态声纹),全局单实例播放。音频存于 Supabase Storage,只信任本站语音桶的地址以防伪造 URL。

### 🎵 文字变洗脑神曲
把输入的文字现场合成为一首神曲,发到房间里全员可点播放 —— **纯前端零后端**,用 Web Audio 合成节拍/贝斯/和弦作伴奏,再用浏览器语音合成把每个字当一个音符卡在拍点上"唱"出来(汉字天然一字一音节,逐字映射到音阶,音高随旋律起伏,反复的旋律动机制造洗脑感)。

点 🎵 按钮选曲风,或用 `/sing 文字`(随机曲风)。共 6 种曲风:

| 曲风 | 特点 |
| --- | --- |
| 🕺 蹦迪DJ | 128 BPM 四踩底鼓 + 锯齿波,快嘴 |
| 🎤 神曲喊麦 | 低音压场,低音区快节奏念白 |
| 🧸 儿歌 | 大调三和弦,高音欢快 |
| 🏮 古风 | 五声音阶,慢板拨弦 |
| 🪩 迪斯科 | 放克贝斯 + offbeat 踩镲 |
| 👾 电玩 | 芯片方波快速琶音 |

> 语音合成依赖系统的中文语音包(桌面 Chrome/Edge、iOS Safari、多数安卓自带)。缺中文语音时只有旋律没有"人声唱字",不影响使用。

## 🛠 技术栈

- **前端**:单文件静态页(`index.html`),原生 HTML/CSS/JS,无构建步骤
- **后端**:[Supabase](https://supabase.com/) — 匿名/密码登录、Postgres + RLS、Realtime(`postgres_changes`)、Storage、Edge Functions
- **实时**:消息与回声走 `postgres_changes` 订阅;在线状态用 `eh_presence` 表 + 心跳(15s)+ 35s 在线窗口
- **音频**:Web Audio API(神曲合成、语音播放)、MediaRecorder(录音)、SpeechSynthesis(逐字演唱)

## 🚀 部署与配置

前端是纯静态页,直接托管 `index.html` 即可。后端需要一个 Supabase 项目,并在页面顶部替换 `SB_URL` / `SB_ANON` 为你自己的地址与 anon key。

Supabase 侧需要准备(简述):

- **数据表**:`eh_users`、`eh_rooms`、`eh_members`、`eh_messages`、`eh_message_echoes`、`eh_presence`,并配好对应 RLS 策略
- **RPC 函数**:`eh_public_recent`(公开房读最近消息)、`eh_find_room_by_code`(按邀请码找房)、`eh_leave_room`(离开/归档公开房)
- **Edge Functions**:`eh-auth`(账号 resolve/register)、`eh-admin-api`(后台接口)
- **Storage 桶**:语音消息需要一个**公开桶 `eh-voice`**,并允许已登录用户上传。桶不存在时发语音会提示"语音存储未配置":

  ```sql
  insert into storage.buckets (id, name, public, file_size_limit, allowed_mime_types)
  values ('eh-voice', 'eh-voice', true, 1048576, array['audio/webm','audio/mp4']);

  create policy "eh voice upload" on storage.objects
    for insert to authenticated
    with check (bucket_id = 'eh-voice');
  ```

## 🔐 后台控制台

`admin.html` 提供管理界面,与前台共用同一套账号:

- **概览**:用户数、各类房间数、消息总数等指标
- **房间与聊天记录**:普通管理员只看公开房,超级管理员可查看全站(含私密房与语音/神曲消息)
- **管理员管理**:超级管理员可任命/撤销管理员、转让超管

> 消息保留策略:公开消息(官方 + 公开房)默认保留 3 个月并每日清理超期;私密房消息永久保留。

## 🛡 安全

- 所有用户可控字段(昵称、emoji、颜色、邀请码、消息内容等)渲染前统一 HTML 转义;`esc()` 转义引号防属性逃逸,颜色走 `safeColor()` 白名单防 CSS 注入
- 语音 / 神曲消息仅信任本站桶前缀或白名单曲风,伪造地址一律按纯文本处理
- 私密房与悄悄话由 RLS 在数据库层拦截,前端再做一次防御
